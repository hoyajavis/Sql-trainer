--- aviation ---
Pass 1: The Foundations (Surface Level)
  Step: Introduction to SELECT Statements
    Lesson: The SELECT statement is the core of SQL. In aviation, you can use it to list airports, aircraft, flights, or crew assignments.
    Snippet: SELECT name, city FROM airports;
    Challenge: As an airline operations manager, you need to quickly identify all available airports within a specific region to plan new routes. Your task is to find all airports located in 'USA'. Retrieve the airport ID, name, city, and country for these airports.
    Solution: SELECT airport_id, name, city, country FROM airports WHERE country = 'USA';
  Step: Selecting Specific Columns
    Lesson: You rarely need all columns. Specify only what you need, like the flight ID and aircraft ID.
    Snippet: SELECT flight_id, aircraft_id FROM flights;
    Challenge: Retrieve only the flight_id and aircraft_id of all flights.
    Solution: SELECT flight_id, aircraft_id FROM flights;
  Step: Filtering Rows with WHERE
    Lesson: The WHERE clause lets you filter records based on specific conditions, like finding a specific aircraft model.
    Snippet: SELECT * FROM aircraft WHERE model = 'Boeing 737';
    Challenge: A maintenance team needs to schedule routine checks for all 'Boeing 737' aircraft in the fleet. To facilitate this, they require a list of all aircraft matching this model. Retrieve the aircraft ID, model, and capacity for all 'Boeing 737' aircraft.
    Solution: SELECT aircraft_id, model, capacity FROM aircraft WHERE model = 'Boeing 737';
  Step: Creating Your First Table
    Lesson: Use CREATE TABLE to define new entities, such as a table for maintenance logs.
    Snippet: CREATE TABLE maintenance_logs (log_id TEXT PRIMARY KEY, aircraft_id TEXT, description TEXT, log_date TEXT, cost REAL);
    Challenge: The maintenance department needs a dedicated table to log all maintenance activities for aircraft. Create a new table named `maintenance_logs` with columns: `log_id` (TEXT PRIMARY KEY), `aircraft_id` (TEXT, FOREIGN KEY to `aircraft`), `description` (TEXT), `log_date` (TEXT), and `cost` (REAL).
    Solution: CREATE TABLE maintenance_logs (
  log_id TEXT PRIMARY KEY,
  aircraft_id TEXT,
  description TEXT,
  log_date TEXT,
  cost REAL,
  FOREIGN KEY (aircraft_id) REFERENCES aircraft(aircraft_id)
);

    Lesson: Use INSERT INTO to add new records, like scheduling a new flight.

    Challenge: The scheduling department has finalized a new flight, 'FL005', from 'LAX' to 'SFO', using aircraft 'AC003', scheduled to depart on '2024-06-10 08:00:00' and arrive on '2024-06-10 09:30:00'. Add this new flight to the system.
    Solution: INSERT INTO flights (flight_id, aircraft_id, departure_airport_id, arrival_airport_id, scheduled_departure, scheduled_arrival) VALUES ('FL005', 'AC003', 'LAX', 'SFO', '2024-06-10 08:00:00', '2024-06-10 09:30:00');

    Lesson: Use DELETE to remove records, such as removing canceled crew assignments.
    Snippet: DELETE FROM crew_assignments WHERE flight_id = 'FL001' AND employee_id = 'EMP007';

Pass 3: Relationships & Security
  Step: Introduction to Table Relationships (INNER JOIN)
    Lesson: JOIN combines rows from two or more tables based on a related column.
    Snippet: SELECT f.flight_id, da.name AS departure_airport, aa.name AS arrival_airport FROM flights f JOIN airports da ON f.departure_airport_id = da.airport_id JOIN airports aa ON f.arrival_airport_id = aa.airport_id;
    Challenge: The operations team needs a comprehensive view of flights along with the model of the aircraft assigned to them. Retrieve the flight_id from the flights table and the model from the aircraft table for all flights using an INNER JOIN.
    Solution: SELECT f.flight_id, a.model FROM flights f JOIN aircraft a ON f.aircraft_id = a.aircraft_id;
  Step: Enforcing Table Architecture (Foreign Keys)
    Lesson: Foreign keys enforce referential integrity between tables.
    Snippet: CREATE TABLE airlines (airline_id INTEGER PRIMARY KEY); CREATE TABLE routes (route_id INTEGER, airline_id INTEGER, FOREIGN KEY (airline_id) REFERENCES airlines(airline_id));
    Challenge: Establish the database layout for airlines and their flight routes. Write the code for an `airlines` table with `airline_id` configured as an in-line INTEGER PRIMARY KEY, plus a `name` column (TEXT NOT NULL UNIQUE). Also, draft a `routes` table with `route_id` (INTEGER), `origin` (TEXT NOT NULL), `destination` (TEXT NOT NULL), `distance` (INTEGER), and `airline_id` (INTEGER). Apply out-of-line constraints assigning `route_id` as the PRIMARY KEY and `airline_id` as a FOREIGN KEY that references `airlines(airline_id)`.
    Solution: CREATE TABLE airlines (airline_id INTEGER PRIMARY KEY, name TEXT NOT NULL UNIQUE); CREATE TABLE routes (route_id INTEGER, origin TEXT NOT NULL, destination TEXT NOT NULL, distance INTEGER, airline_id INTEGER, PRIMARY KEY (route_id), FOREIGN KEY (airline_id) REFERENCES airlines(airline_id));
  Step: Inline Data Constraints (NOT NULL, UNIQUE, CHECK)

Pass 4: Advanced Queries & Subqueries
  Step: Including Unmatched Records (LEFT JOIN)
    Lesson: LEFT JOIN returns all records from the left table, and matched records from the right.
    Snippet: SELECT a.name, COUNT(f.flight_id) FROM airports a LEFT JOIN flights f ON a.airport_id = f.departure_airport_id GROUP BY a.name;
    Challenge: Airport authorities want to analyze traffic patterns by identifying which airports handle the most departures. This data is crucial for resource allocation and expansion plans. Retrieve the airport name and the count of flights departing from each airport. (Only include airports that have at least one departing flight). Name the result column 'departure_count'.
    Solution: SELECT ap.name, COUNT(f.flight_id) AS departure_count FROM airports ap JOIN flights f ON ap.airport_id = f.departure_airport_id GROUP BY ap.name ORDER BY departure_count DESC;
  Step: Multi-Table Chains
    Lesson: You can join multiple tables to get richer insights.
    Snippet: SELECT DISTINCT ca.employee_id FROM crew_assignments ca JOIN flights f ON ca.flight_id = f.flight_id JOIN airports da ON f.departure_airport_id = da.airport_id JOIN airports aa ON f.arrival_airport_id = aa.airport_id WHERE da.city = 'London' OR aa.city = 'London';
    Challenge: For a user-friendly flight schedule display, the system needs to show the full names of the departure and arrival airports instead of just their IDs. This improves readability for passengers. Retrieve the flight ID, the name of the departure airport, and the name of the arrival airport. Name the result column 'departure_airport_name'.
    Solution: SELECT f.flight_id, da.name AS departure_airport_name, aa.name AS arrival_airport_name FROM flights f JOIN airports da ON f.departure_airport_id = da.airport_id JOIN airports aa ON f.arrival_airport_id = aa.airport_id;
  Step: Finding Missing Data (IS NULL)
    Lesson: Use IS NULL to find left table records that have no match in the right table.
    Snippet: SELECT a.name FROM airports a LEFT JOIN flights f ON a.airport_id = f.departure_airport_id WHERE f.flight_id IS NULL;
    Challenge: Find airports that have no departures. Select airport_id and name using a LEFT JOIN with flights.
    Solution: SELECT a.airport_id, a.name FROM airports a LEFT JOIN flights f ON a.airport_id = f.departure_airport_id WHERE f.flight_id IS NULL;
  Step: Filtering using Subqueries (Left Join/Subquery combos)
    Lesson: Find records that do not match a certain set using NOT IN.
    Snippet: SELECT name FROM airports WHERE airport_id NOT IN (SELECT DISTINCT departure_airport_id FROM flights);
    Challenge: The network planning team wants to identify airports that currently have no scheduled departing flights. This could indicate underutilized infrastructure or potential route cancellations. Retrieve the name and city of airports with no departing flights.
    Solution: SELECT name, city FROM airports WHERE airport_id NOT IN (SELECT DISTINCT departure_airport_id FROM flights);
  Step: Advanced Subquery Comparisons
    Lesson: A subquery can be used in the WHERE clause to compare against dynamic values.
    Snippet: SELECT * FROM aircraft WHERE capacity > (SELECT AVG(capacity) FROM aircraft);
    Challenge: Find aircraft whose capacity is higher than the average capacity among all aircraft.
    Solution: SELECT * FROM aircraft WHERE capacity > (SELECT AVG(capacity) FROM aircraft);
  Step: Conditional Logic with CASE WHEN
    Lesson: CASE allows for conditional logic in your SELECT statements.
    Snippet: SELECT aircraft_id, CASE WHEN capacity > 200 THEN 'Large' ELSE 'Standard' END FROM aircraft;
    Challenge: Select aircraft_id and a 'size_category' column which is 'Large' if capacity > 200 else 'Standard'. Do this for aircraft.
    Solution: SELECT aircraft_id, CASE WHEN capacity > 200 THEN 'Large' ELSE 'Standard' END AS size_category FROM aircraft;
  Step: Subqueries in the SELECT Clause
    Lesson: You can also put subqueries directly in the SELECT list to calculate values row by row.
    Snippet: SELECT name, (SELECT COUNT(*) FROM aircraft) AS total FROM airports;
    Challenge: Select airport names and the total number of aircraft in the entire system as a column aliased as 'total_aircraft_db'.
    Solution: SELECT name, (SELECT COUNT(*) FROM aircraft) AS total_aircraft_db FROM airports;
Pass 5: Masterclass Architecture & Advanced Features
  Step: Composite Primary Keys and Out-of-Line Constraints
    Lesson: Composite keys use multiple columns to uniquely identify a record.
    Snippet: CREATE TABLE terminals (terminal_id INTEGER, hub_id INTEGER, PRIMARY KEY (terminal_id, hub_id));
    Challenge: Establish a system for airports and their terminals. Set up an `airports_hub` table with an in-line `hub_id` (INTEGER PRIMARY KEY), `code` (TEXT NOT NULL UNIQUE), and `city` (TEXT). Follow this by creating a `terminals` table carrying `terminal_id` (INTEGER), `gate_count` (INTEGER), `is_active` (INTEGER DEFAULT 1), and `hub_id` (INTEGER). For the `terminals` table, declare out-of-line conditions to make `terminal_id` the PRIMARY KEY, a CHECK that `is_active IN (0, 1)`, and a FOREIGN KEY mapping `hub_id` to `airports_hub(hub_id)`.
    Solution: CREATE TABLE airports_hub (hub_id INTEGER PRIMARY KEY, code TEXT NOT NULL UNIQUE, city TEXT); CREATE TABLE terminals (terminal_id INTEGER, gate_count INTEGER, is_active INTEGER DEFAULT 1, hub_id INTEGER, PRIMARY KEY (terminal_id), CHECK (is_active IN (0, 1)), FOREIGN KEY (hub_id) REFERENCES airports_hub(hub_id));
  Step: Advanced Schema Planning
    Lesson: Combine DEFAULT, CHECK, and FOREIGN KEY for robust schemas.
    Snippet: CREATE TABLE repair_logs (log_id BIGINT, hours_logged DECIMAL(5,1) NOT NULL, CHECK (hours_logged > 0));
    Challenge: Structure a database for aircraft repairs. Set up a `parts_catalog` table using an in-line `part_no` (VARCHAR(50) PRIMARY KEY) and `description` (VARCHAR(255) NOT NULL). Draft a `repair_logs` table equipped with `log_id` (BIGINT), `hours_logged` (DECIMAL(5,1) NOT NULL), `repair_date` (DATE), and `part_no` (VARCHAR(50)). Supply out-of-line constraints in `repair_logs`: a PRIMARY KEY for `log_id`, a CHECK ensuring `hours_logged > 0.0`, and a FOREIGN KEY pointing `part_no` to `parts_catalog(part_no)`. Separate the statements with a semicolon.
    Solution: CREATE TABLE parts_catalog (part_no VARCHAR(50) PRIMARY KEY, description VARCHAR(255) NOT NULL); CREATE TABLE repair_logs (log_id BIGINT, hours_logged DECIMAL(5,1) NOT NULL, repair_date DATE, part_no VARCHAR(50), PRIMARY KEY (log_id), CHECK (hours_logged > 0.0), FOREIGN KEY (part_no) REFERENCES parts_catalog(part_no));
  Step: Speeding Up Lookups (CREATE INDEX)
    Lesson: Indexes speed up retrieval operations on a table.
    Snippet: CREATE INDEX idx_aircraft_model ON aircraft (model);
    Challenge: Create an index named `idx_aircraft_model` on the `model` column in the `aircraft` table.
    Solution: CREATE INDEX idx_aircraft_model ON aircraft (model);
  Step: Saving Complex Queries (CREATE VIEW)
    Lesson: Views are virtual tables based on the result of an SQL statement.
    Snippet: CREATE VIEW usa_airports AS SELECT * FROM airports WHERE country = 'USA';
    Challenge: Create a view named `usa_airports` to see only airports where country = 'USA'.
    Solution: CREATE VIEW usa_airports AS SELECT * FROM airports WHERE country = 'USA';
  Step: Building Custom Views
    Lesson: Views help simplify complex queries or restrict data access.
    Snippet: CREATE VIEW grounded_aircraft AS SELECT * FROM aircraft WHERE status = 'Grounded';

    Solution: CREATE VIEW grounded_aircraft AS SELECT * FROM aircraft WHERE status = 'Grounded';
  Step: Altering Existing Views
    Lesson: To change or delete a view, you drop it.
    Snippet: DROP VIEW grounded_aircraft;
    Challenge: Drop the `grounded_aircraft` view.
    Solution: DROP VIEW grounded_aircraft;
  Step: The Recreation Pattern for Complex Modfications
    Lesson: SQLite limits ALTER TABLE, so you sometimes recreate, copy data, drop, and rename.
    Snippet: CREATE TABLE a_new (id TEXT); INSERT INTO a_new SELECT airport_id FROM airports; DROP TABLE airports;
    Challenge: Recreate `airports_new` as `airport_id` TEXT PRIMARY KEY, `name` TEXT. Insert from `airports`. Drop `airports`. Rename `airports_new` to `airports`.
    Solution: CREATE TABLE airports_new (airport_id TEXT PRIMARY KEY, name TEXT); INSERT INTO airports_new (airport_id, name) SELECT airport_id, name FROM airports; DROP TABLE airports; ALTER TABLE airports_new RENAME TO airports;
  Step: Common Table Expressions (CTEs/WITH clauses)

  Step: Window Functions
    Lesson: Window functions calculate values across a set of table rows without grouping them.
    Snippet: SELECT name, ROW_NUMBER() OVER(ORDER BY city ASC) AS rank FROM airports;
    Challenge: Select airport names and rank them alphabetically by city using ROW_NUMBER. Give the rank column the alias 'rank'.
    Solution: SELECT name, ROW_NUMBER() OVER(ORDER BY city ASC) AS rank FROM airports;

    Challenge: Assemble the layout for condominium buildings and their units.
